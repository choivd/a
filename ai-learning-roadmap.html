Phase 01
Mental Models
4–6 weeks
Core

How language models actually work — not math, but mechanics. You need to think in the right vocabulary before you can design around these systems.

Core Topics

  • ▸ What a language model is — tokens, context windows, probability
  • ▸ Why models hallucinate — and what actually causes it
  • ▸ How temperature and parameters shape outputs
  • ▸ Base models vs fine-tuned models vs RLHF
  • ▸ What an API is and how data flows through a request
Resources
Generative AI with Large Language Models
O'Reilly / DeepLearning.ai — Start here. Conceptual, not code-heavy.
"How LLMs Work" by Andrej Karpathy
YouTube — Free. The clearest plain-language explanation that exists.
Anthropic's Model Documentation
docs.anthropic.com — Free. Read how Claude is designed to behave.
Hands-On Exercises
01 Open the Anthropic or OpenAI playground. Ask the same question 5 different ways. Document what shifts in the output and why.
02 Adjust temperature from 0 to 1 on a creative prompt, then a factual one. Observe what changes at each extreme.
03 Intentionally try to make the model hallucinate. Then figure out exactly what triggered it. Understanding failure is understanding the system.
04 Count tokens with tiktokenizer.vercel.app. Get a feel for how the model actually sees text — not words, but chunks.
Phase Artifact — Build One of These
Every phase ends with a working artifact. Tiny is fine. The goal is to stop being only a consumer of models.
Prompt observation journal Tokenizer visual explainer Playground experiment log with annotations
Phase 02
Prompt Engineering & Context Design
4–6 weeks
Core

The hands-on craft. How to structure inputs so models produce reliable, high-quality outputs. This is where your systems thinking immediately applies.

Core Topics

  • ▸ Prompt structure — roles, instructions, constraints, examples
  • ▸ Chain-of-thought and reasoning techniques
  • ▸ Few-shot vs zero-shot prompting and when to use each
  • ▸ Context window management — what to include, what to cut
  • ▸ System prompts vs user prompts vs memory
  • ▸ Output formatting and structured responses
Resources
Prompt Engineering for Developers
DeepLearning.ai — Free. Short, practical, essential. Has interactive notebooks.
Anthropic Prompt Engineering Guide
docs.anthropic.com — Free. Surprisingly deep.
Prompt Engineering (O'Reilly)
O'Reilly — Multiple titles available. Search and pick the most recent.
Hands-On Exercises
01 Pick one real task you do weekly. Write three system prompts with different structures. Compare outputs and note what changed and why.
02 Take a prompt with bad output. Diagnose why — instruction clarity? Missing examples? Ambiguous constraints? Fix one variable at a time.
03 Add "think step by step" to a complex prompt. Compare to the same prompt without it. This is chain-of-thought in action.
04 Write a deliberately terrible prompt. Then rewrite it systematically, documenting every change and the reasoning behind each decision.
Phase Artifact — Build One of These
Build a workflow you personally use weekly. If you use it repeatedly, you learn much faster.
Research synthesizer UX critique assistant Meeting summarizer Design pattern classifier Portfolio reviewer
Phase 03
Data & Knowledge Systems
4–6 weeks
Data & Knowledge

One of the biggest gaps in most AI learning paths. AI systems are increasingly data systems. This area aligns extremely well with your systems-thinking background — taxonomy design, knowledge representation, retrieval quality.

Core Topics

  • ▸ Embeddings — how meaning becomes numbers
  • ▸ Vector databases — how AI stores and retrieves knowledge
  • ▸ Chunking strategies — how documents get split for retrieval
  • ▸ Metadata design and taxonomy thinking
  • ▸ Retrieval quality — precision, recall, relevance
  • ▸ Knowledge graphs and ontologies
  • ▸ Synthetic data generation
  • ▸ Annotation, labeling, and evaluation datasets
Resources
LangChain & Vector Databases in Production
O'Reilly — Covers RAG and retrieval architecture conceptually.
Building RAG Applications
DeepLearning.ai — Free. Practical and conceptual with hands-on notebooks.
Knowledge Graphs (O'Reilly)
O'Reilly — For the structural, taxonomic thinking layer.
Hands-On Exercises
01 Take one document. Chunk it three ways — by sentence, by paragraph, by semantic topic. Ask the same question against each. Compare what gets retrieved.
02 Design a tagging schema and taxonomy for a knowledge base you actually use. Think about relationships, hierarchy, and edge cases.
03 Manually create a small eval dataset: 20 question-answer pairs from a document. This is the foundation of evaluation thinking.
04 Test retrieval quality qualitatively — ask 10 questions, score whether the retrieved context actually answers each one. Where does it fail and why?
Phase Artifact — Build One of These
Tagging schema for your Obsidian vault Knowledge taxonomy with relationship map Chunking strategy comparison document Eval dataset (20+ Q&A pairs)
Phase 04
Technical Fluency for AI Builders
6–8 weeks
Technical

You don't need to become a software engineer. But people with disproportionate leverage now are technical enough to build, orchestrate, and automate independently. Without this, you remain dependent on engineers for execution.

Core Topics

  • ▸ JSON — reading and writing structured data
  • ▸ REST APIs — how services communicate
  • ▸ Python basics — enough to prototype and test ideas
  • ▸ Webhooks and authentication concepts
  • ▸ Parsing and transforming structured AI outputs
  • ▸ Local vs cloud execution
  • ▸ Git / GitHub — version control basics
  • ▸ AI SDK basics — calling models programmatically

Key distinction: you need operational fluency, not computer science. These are very different things.

Resources
Python for Everybody
O'Reilly / Coursera — The gentlest Python on-ramp. Start here.
APIs for Beginners (freeCodeCamp)
YouTube — Free. Clear, practical introduction to how APIs work.
Anthropic & OpenAI API Quickstarts
Official docs — Free. Getting-started guides with real, runnable code.
Hands-On Exercises
01 Call the Anthropic or OpenAI API directly from code — not a UI. See the raw request and response JSON. Read it.
02 Build a tiny Streamlit app — one text input, one AI call, one output. Deploy it. Show it to one person.
03 Connect two services — Airtable or Notion to an AI call. Parse the output and write it somewhere else.
04 Build a simple agent workflow in code: one tool call, one decision, one output. Understand what "orchestration" actually means technically.

Phase 05
AI Systems Architecture
6–8 weeks
Core

How multiple AI components fit together into a working system. Your design instinct for relationships between building blocks becomes a direct advantage here.

Core Topics

  • ▸ RAG — giving AI access to real, up-to-date data
  • ▸ AI agents and multi-step reasoning flows
  • ▸ Memory — short-term, long-term, external stores
  • ▸ Tool use — how AI calls external functions
  • ▸ Chaining models — orchestrating multiple AI calls
  • ▸ Guardrails, fallbacks, and safety layers
  • ▸ Latency, cost, and reliability trade-offs in production
Resources
Building Systems with the ChatGPT API
DeepLearning.ai — Free. Great for understanding chaining and workflows.
Designing Large Language Model Applications
O'Reilly — Systems-focused, less code. Closest to your angle.
AI Engineering — Chip Huyen
O'Reilly — The most comprehensive book on production AI systems.
Hands-On Exercises
01 Use Flowise or LangFlow (free, visual, no-code) to build a RAG pipeline. Upload a document, ask questions. Watch how retrieval actually works.
02 Map an existing AI product as a system diagram — inputs, outputs, memory, tools, guardrails. Identify what you don't know and research it.
03 Design a multi-step workflow on paper — what passes forward at each step, where things could break. Then build it with a no-code tool.
04 Find a public AI system that fails in an interesting way. Hypothesize the architecture and where the failure originates. Write it up.
Phase 06
Evaluation & Quality Thinking
Ongoing from Phase 02
Core

The layer most people skip. How do you actually know if an AI system is working? This is where rigor separates serious practitioners from tinkerers — and where your design instinct for quality translates directly.

Core Topics

  • ▸ Evaluation metrics — defining what "good output" actually means
  • ▸ Red-teaming — systematically finding failure modes
  • ▸ Human feedback loops and RLHF concepts
  • ▸ A/B testing AI outputs with scoring rubrics
  • ▸ Building evals — writing repeatable tests for AI behavior
  • ▸ Monitoring and regression tracking in production
Resources
Evaluating and Debugging Generative AI
DeepLearning.ai — Free. Exactly this topic.
AI Engineering — Evals Chapter (Chip Huyen)
O'Reilly — The evals chapter alone is worth the book.
Anthropic's Evals Framework
GitHub / docs — Free. See how professional evaluation actually works.
Hands-On Exercises
01 Pick a prompt from Phase 02. Write 10 test cases — inputs with expected outputs. Run them. Score each one. This is an eval.
02 Red-team your own prompt system for 30 minutes — edge cases, weird inputs, adversarial phrasing. Document every failure mode.
03 Run two prompt versions on 10 identical inputs. Build a scoring rubric. Judge which performs better and write up exactly why.
04 Define what "good" looks like for one specific AI task in concrete, measurable terms. This is harder than it sounds — and it's the most important skill in this phase.
Phase 07
Human Systems & Organizational Dynamics
Parallel / Long-term
Ongoing

This may become your strongest long-term advantage. Most AI practitioners are technically capable but weak at organizational behavior, adoption friction, and human coordination. Enterprises struggle with exactly these things.

Core Topics

  • ▸ AI governance and policy within organizations
  • ▸ Change management — why AI adoption fails and how to fix it
  • ▸ AI UX trust models — how humans calibrate trust in AI outputs
  • ▸ Human-in-the-loop system design
  • ▸ Workflow redesign around AI capabilities
  • ▸ Decision-support systems — when AI aids vs when it decides
  • ▸ Socio-technical systems — human and technical together
  • ▸ Incentive mapping — why people resist or adopt new systems

A designer who understands AI systems + human systems + organizational systems simultaneously is very hard to replace. This is where "designer" evolves into "systems architect."

Resources
Thinking in Systems — Donella Meadows
Book — The foundational text for systems thinking applied to human organizations.
AI and the Future of Work (O'Reilly)
O'Reilly — Organizational and human dimensions of AI adoption.
Human + Machine — Daugherty & Wilson
Book — How AI and human roles actually get redesigned together in organizations.
Hands-On Exercises
01 Map an AI adoption failure you've observed. What were the human, organizational, and technical factors? Where did the system break?
02 Design a human-in-the-loop workflow for a high-stakes decision. Where does AI assist? Where does a human have final say? Justify each boundary.
03 Interview three people about how they actually use AI at work vs how they're supposed to. Document the gap. That gap is your design space.
04 Write a one-page AI governance brief for a fictional org introducing AI to a sensitive workflow. What rules, checks, and human roles would you define?
