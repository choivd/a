Week
01
Core
Mental Models
You likely know the basics. Use this week to fill specific gaps and lock in precise vocabulary — tokens, context windows, temperature, RLHF. The goal is fluency, not discovery.
Skip if you know it
✗ What an LLM is generally
✗ That hallucinations happen
✗ Basic transformer overview
✗ What embeddings are roughly
Do this week
▸ Nail the exact definition of a token and context window
▸ Understand WHY hallucinations happen mechanically
▸ Know what temperature actually controls
▸ Read Karpathy's LLM explainer in full
Week Artifact
→ Write a 200-word plain-language explanation of how LLMs work. If you can't, that's your gap map.
→ List 3 things that still feel fuzzy — these become Round 2 priorities.
Round 2 flag RLHF mechanics, fine-tuning vs prompting trade-offs, attention mechanism depth
Week
02
Core
Prompt Engineering & Context Design
Stop reading about prompting — just prompt. One real problem, multiple approaches, document what breaks. You'll learn more from 3 hours of active experimentation than 5 hours of reading.
Skip if you know it
✗ "Be specific in your prompts"
✗ Basic prompt tips articles
✗ Watching prompting demos
Do this week
▸ Build one system prompt for a real task you do
▸ Rewrite it 4 different ways — compare outputs
▸ Add chain-of-thought. Observe the difference.
▸ Hit a context window limit intentionally
▸ Read Anthropic's prompt engineering guide
Week Artifact
→ One working prompt system for something you use weekly. Document what you changed and why.
Round 2 flag Multi-turn memory management, structured output design, advanced few-shot strategies
Week
03
Data
Data & Knowledge Systems
Probably your biggest real gap. Embeddings, chunking, retrieval — these sound familiar but most designers can't explain them precisely. This week is about building actual intuition, not just vocabulary.
Skip if you know it
✗ "Embeddings turn text into numbers"
✗ High-level RAG overview videos
Do this week
▸ Read one solid explainer on RAG end-to-end
▸ Understand chunking — why it matters and what bad chunking causes
▸ Understand what "retrieval precision" means in practice
▸ Design a taxonomy for one knowledge base you actually use
Week Artifact
→ A tagging schema or taxonomy map for a real knowledge base — your notes, your work, anything real.
Round 2 flag Vector database implementation, chunking strategy comparison, knowledge graph design, eval datasets
Week
04
Technical
Technical Fluency for AI Builders
The most likely uncomfortable week. Don't aim to learn Python — aim to make one real API call and read the JSON response. That's the whole goal. You need operational fluency, not programming depth.
Skip if you know it
✗ What an API is conceptually
✗ What JSON looks like
✗ Git basics
Do this week
▸ Make one real API call to Anthropic or OpenAI
▸ Read the raw JSON response — understand its structure
▸ Build the simplest possible Streamlit app
▸ Push both files of your learning journal to GitHub
Week Artifact
→ Your learning journal live on GitHub Pages. This forces you to actually use Git and GitHub — the technical fluency goal in one real task.
Round 2 flag Python scripting, webhooks, authentication flows, connecting multiple services, agent workflows in code
Week
05
Core
AI Systems Architecture
This is where your systems thinking kicks in. Don't try to build anything complex — try to map something that already exists. Understanding how the pieces connect matters more than building this week.
Skip if you know it
✗ General overview videos on RAG
✗ "AI agents are autonomous" explainers
Do this week
▸ Use Flowise or LangFlow (no-code) to build one RAG pipeline
▸ Map an AI product you use as a system diagram
▸ Identify: where is the memory? where are the guardrails?
▸ Read the architecture chapter of AI Engineering (Chip Huyen)
Week Artifact
→ One system diagram of a real AI product — inputs, outputs, memory, tools, failure points. Post it on your journal.
Round 2 flag Multi-agent orchestration, memory architecture patterns, latency/cost trade-offs, building agents in code
Week
06
Core
Evaluation & Quality Thinking
Most people skip this entirely. Don't. Even a rough eval on something you built earlier will teach you more about AI systems than another week of reading. The act of defining "good" is the skill.
Skip if you know it
✗ "Evals are important" articles
✗ RLHF conceptual overview (covered in Week 1)
Do this week
▸ Write 10 test cases for the prompt you built in Week 2
▸ Score each output against a rubric you define
▸ Red-team your own prompt — try to break it in 5 ways
▸ Define "good output" for one task in concrete, measurable terms
Week Artifact
→ A simple eval: 10 test cases, a scoring rubric, and your results. Even a table in a text file counts.
Round 2 flag Automated eval pipelines, LLM-as-judge patterns, regression testing, production monitoring
Week
07
Ongoing
Human Systems & Organizational Dynamics
Your likely strongest long-term advantage as a designer. This week is about framing your existing knowledge — you understand human systems more than you think. The goal is connecting that to AI contexts.
Skip if you know it
✗ General change management theory
✗ UX research basics
Do this week
▸ Map one AI adoption failure you've seen — human, org, and technical factors
▸ Design a human-in-the-loop workflow for one high-stakes AI task
▸ Read one case study on AI deployment that failed due to human factors
▸ Write your own "what I now know" reflection — the full 7 weeks
Week Artifact
→ A "Round 1 complete" post on your learning journal. What clicked, what surprised you, what you're going deeper on in Round 2.
Round 2 flag AI governance frameworks, trust calibration models, socio-technical system design, organizational AI strategy
